# Washnet - Complete Laundry Management System

## System Overview

Washnet is a comprehensive Laundry Point-of-Sale (POS) and Management Web Application built with Django that orchestrates the complete laundry lifecycle from customer ordering to final delivery. The system features role-based access control with distinct workflows for Customers, Employees, Riders, and Administrators, plus an AI-powered chatbot for customer support.

## Technical Architecture

### Core Technologies
- Backend: Django 6.0.1 with Django REST Framework
- Database: SQLite (development) / PostgreSQL (production)
- Frontend: Django Templates with custom CSS (Claymorphism design)
- AI: Google Gemini API with function calling
- Authentication: Django Auth with custom User model
- Payments: Cash, GCash, PayPal integration
- QR Codes: Receipt generation and verification

### Key Features
- 10-Stage Order Status System
- Load-based Pricing (max 7kg per load)
- Real-time Order Tracking
- Weather Forecast Integration
- AI Chatbot with Role-Specific Tools
- Comprehensive Analytics Dashboard
- Mobile-Responsive Design

## Complete Order Flow Logic

### Order Lifecycle (10 Status States):

1. EXPECTING_DROP_OFF → Customer created drop-off order, awaiting arrival
2. PENDING_ACCEPTANCE → Delivery order waiting for rider assignment
3. RIDER_ACCEPTED → Rider accepted pickup job, en route to customer
4. PICKED_UP → Rider collected laundry, transporting to shop
5. AT_SHOP → Laundry arrived at shop, waiting for processing
6. PROCESSING → Employee actively washing/drying laundry
7. READY_FOR_DELIVERY → Processing complete, ready for rider pickup
8. OUT_FOR_DELIVERY → Rider collected from shop, delivering to customer
9. COMPLETED → Order successfully delivered to customer
10. CANCELLED → Order cancelled by customer or system

### Business Logic Rules:

#### Ownership & Assignment
- Employee Ownership: Walk-in orders belong to creating employee
- Rider Assignment: First-come, first-served for available jobs
- Admin Override: Administrators can modify any assignment

#### Status Transition Rules
- Employee Restrictions: Cannot set rider-specific statuses (PENDING_ACCEPTANCE, RIDER_ACCEPTED, PICKED_UP, OUT_FOR_DELIVERY)
- Rider Restrictions: Cannot set shop processing statuses (AT_SHOP, PROCESSING, READY_FOR_DELIVERY)
- Payment Requirements: GCash orders require admin verification before completion

#### Load Processing Logic
- Maximum Load: 7kg per physical load
- Service Pricing: Per kilogram for laundry services
- Product Pricing: Per piece for add-ons
- Load Separation: Customers can create multiple loads with different configurations
